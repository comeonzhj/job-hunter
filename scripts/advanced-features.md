# 🚀 进阶玩法 — 让长期求职任务更有生命力

## 1. 文档即界面（已实现）

**核心理念**: 飞书文档不只是信息载体，是交互界面。

用户在文档里打勾 = 反馈。系统轮询文档 = 采集信号。
不需要额外的表单、链接、工具，文档本身就是产品。

**为什么这很特别？**
- 用户操作成本为零（已经在看文档了）
- 反馈即时可见（打勾后自己能看到）
- 上下文完整（看着岗位描述做决策）
- 可追溯（文档保留完整历史）

---

## 2. 沉默信号学习

**问题**: 用户不回复 = 没有反馈？不对，沉默本身就是信号。

**设计**:
- 用户 72h 不碰某个岗位 → 弱负信号（-0.5）
- 用户连续 3 批不碰某类岗位 → 强负信号（-2），降低该类推荐权重
- 但不等于排除 → 因为可能只是没看到

**为什么这很重要？**
- 传统推荐系统只学"显式反馈"，这里"隐式反馈"也被捕获
- 系统不会因为用户懒得回复就停止进化
- 沉默信号的衰减设计防止过度推断

## 3. 偏好漂移检测

**问题**: 用户的求职方向会变。第一个月想做 AI Agent，第二个月可能转向 AI infra。

**设计**:
```json
{
  "preference_trajectory": {
    "month_1": {"top_skill": "LLM", "top_industry": "AI"},
    "month_2": {"top_skill": "Kubernetes", "top_industry": "Cloud"},
    "drift_detected": true,
    "drift_direction": "AI应用 → AI基础设施"
  }
}
```

**触发条件**:
- 最近 10 个正向信号的特征分布与前 30 天有显著差异
- 检测到漂移 → 发送确认消息："发现你的偏好似乎在变化，是否要调整搜索方向？"

**为什么这很酷？**
- 系统有"记忆"也有"觉察"
- 不会锁定在初始配置里
- 主动感知变化，而不是被动等用户说

## 4. 市场情报模式

**超越"找工作"**: 不只是被动接收市场信息，主动分析市场趋势。

**数据来源**: 搜索结果本身就是市场数据。

**可分析维度**:

| 维度 | 分析方式 | 输出 |
|------|---------|------|
| 薪资趋势 | 同岗位 30 天薪资变化 | "AI Agent 岗位薪资中位数较上月 ↑8%" |
| 招聘热度 | 关键词搜索结果数量变化 | "LLM 相关岗位本周新增 120 个，环比 ↑15%" |
| 公司动态 | 某公司连续发布多个岗位 | "XX 科技本周新发 5 个 AI 岗位，可能在组建新团队" |
| 竞争指数 | 岗位发布时长 vs 是否关闭 | "热门岗位平均存活时间仅 3 天，建议快速投递" |
| 技能需求 | JD 高频关键词变化 | "RAG 出现频率较上月 ↑30%，成为热门加分项" |

**输出形式**: 每周市场情报报告（飞书文档）

**为什么这很有价值？**
- 求职不只是"投简历"，是"理解市场"
- 帮用户做更好的职业决策
- 让用户觉得这个系统"不只是工具，是顾问"

## 5. 周报摘要

**频率**: 每周日生成

**内容结构**:

```markdown
# 📊 求职周报 — 第 3 周

## 本周概览
- 新增岗位: 45 个
- 推荐投递: 12 个
- 你标记感兴趣: 5 个
- 已投递: 3 个
- 自动归档: 8 个

## 你的求职漏斗
搜索 45 → 推荐 12 → 感兴趣 5 → 投递 3 → 面试 ? → Offer ?

## 市场洞察
- AI Agent 岗位薪资中位数: 42K (↑5%)
- 本周新增热门公司: XX 科技、YY 智能
- 热门技能: RAG (+25%), Agent Framework (+18%)

## 偏好模型更新
- 系统已学习到你偏好: 大模型应用 > 基础设施
- 排除规则新增: "外包"（来自你的 3 次标记）
- 评分精度: ↑12%（相比上周）

## 下周建议
- 关注 XX 科技的新团队组建动态
- 考虑补充 "向量数据库" 技能（市场需求 ↑30%）
- 建议投递速度加快（热门岗位存活时间缩短）
```

## 6. 面试准备模式

**触发**: 用户在文档中勾选岗位 + 评论"已投"

**自动激活**:
1. 生成该公司的面试准备文档
2. 基于 JD 预测面试问题
3. 提供回答要点和项目经历映射
4. 追踪面试进度

**面试准备文档结构**:
```markdown
# 🎯 面试准备 — {company} {title}

## 公司背景
- 融资阶段 / 规模 / 核心业务 / 近期动态

## 岗位解读
- JD 关键要求拆解
- 隐含需求分析

## 预测问题 (基于 JD 生成)
1. 技术题: "请描述你做过的 RAG 系统..."
2. 场景题: "如果让你设计一个多 Agent 协作框架..."
3. 行为题: "描述一次你解决复杂技术问题的经历..."

## 你的经历映射
- 经历 A → 匹配问题 1, 3
- 经历 B → 匹配问题 2

## 薪资谈判参考
- 市场薪资区间: 35-55K
- 该公司薪资水平: 中等偏上
- 建议报价: 45K（基于你的经验和市场数据）
```

## 8. 推荐网络发现

**想法**: 分析用户的社交网络，发现内推机会。

**实现思路**:
- 用户提供通讯录 / 飞书组织架构
- 匹配目标公司是否有认识的人
- 输出: "你的前同事张三在 XX 科技，可以考虑内推"

**注意**: 这需要额外的权限和数据，作为可选模块。

## 9. 求职进度看板

**可视化**: 在飞书多维表格中维护求职进度。

**字段**:
| 公司 | 岗位 | 薪资 | 状态 | 投递日期 | 面试日期 | 结果 | 备注 |
|------|------|------|------|---------|---------|------|------|
| XX科技 | AI Agent工程师 | 40K | 已投递 | 01-15 | 01-20 | 待定 | 二面中 |

**状态流转**: 新发现 → 感兴趣 → 已投递 → 面试中 → Offer/拒绝

**自动化**: 当用户标记"已投"时，自动在看板中创建记录。

## 10. 激励机制

**问题**: 长期求职容易倦怠。

**设计**:
- 🏆 连续 7 天查看报告: "求职达人" 徽章
- 📈 本周投递 5+ 个: "行动力满分"
- 🎯 收到面试邀请: "初筛通过"
- 💡 偏好模型准确率 > 80%: "系统已深度理解你"

**不是游戏化，是正反馈**: 让用户觉得"有进展"而不是"在等待"。
